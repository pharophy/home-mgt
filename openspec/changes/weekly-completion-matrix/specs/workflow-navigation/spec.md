## MODIFIED Requirements

### Requirement: The client SHALL provide a single primary workflow surface for weekly task completion
The system SHALL organize the parent/admin experience around one primary weekly completion matrix surface instead of requiring top-level navigation between parallel parent/admin workflows for the core task-tracking job.

#### Scenario: Parent/admin user opens the app
- **WHEN** a parent/admin user enters the product for normal weekly task tracking
- **THEN** the system opens the weekly completion matrix as the primary surface instead of requiring the user to choose among multiple top-level workflow destinations first
